# Google Cloud Console 설정 확인 결과

## ✅ 올바르게 설정된 항목

### 1. API 키 제한 설정
- ✅ **애플리케이션 제한사항**: "웹사이트" 선택됨
- ✅ **웹사이트 제한사항**: 다음이 올바르게 추가되어 있음
  - `http://127.0.0.1:3000/*`
  - `http://localhost:3000/*`
- ✅ **API 제한사항**: "키 제한" 선택됨
- ✅ **Maps JavaScript API**: API 제한사항에 포함되어 있음

### 2. 백엔드 서버
- ✅ 백엔드 서버가 실행 중 (포트 3001)

## ⚠️ 확인이 필요한 항목

### 1. 결제 계정 연결 확인

현재 "Pay as you go" 요금제를 사용 중이지만, **결제 계정이 실제로 연결되어 있는지** 확인이 필요합니다.

**확인 방법:**
1. Google Cloud Console에서 **"결제"** 메뉴로 이동
2. **"결제 계정"** 섹션 확인
3. 결제 계정이 연결되어 있는지 확인
   - 연결되어 있지 않다면: "결제 계정 추가" 클릭하여 연결

**중요**: Google Maps API는 결제 계정이 연결되어 있어야 작동합니다. 무료 할당량이 제공되지만, 결제 계정 자체는 필요합니다.

### 2. Maps JavaScript API 활성화 확인

API 키 제한사항에 Maps JavaScript API가 포함되어 있지만, **API 자체가 활성화되어 있는지** 확인이 필요합니다.

**확인 방법:**
1. Google Cloud Console에서 **"API 및 서비스" > "사용 설정된 API 및 서비스"** 메뉴로 이동
2. **"Maps JavaScript API"**가 목록에 있는지 확인
3. 없다면:
   - **"API 및 서비스" > "라이브러리"** 메뉴로 이동
   - **"Maps JavaScript API"** 검색
   - **"사용"** 버튼 클릭

### 3. API 키가 올바른 프로젝트에 속해 있는지 확인

현재 프로젝트 이름이 "onepiece"로 보입니다. API 키가 이 프로젝트에서 생성되었는지 확인하세요.

## 🔍 추가 확인 사항

### 환경 변수 확인

프론트엔드 `.env.local` 파일의 API 키가 Google Cloud Console에 표시된 API 키와 일치하는지 확인:

```env
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY=여기에_API_키_입력
```

**API 키 확인 방법:**
1. Google Cloud Console > "사용자 인증 정보" > API 키 클릭
2. "키 표시" 버튼 클릭하여 전체 API 키 확인
3. `.env.local` 파일의 키와 비교

### 설정 변경 후 대기 시간

Google Cloud Console에서 설정을 변경한 경우:
- **5-10분 정도 대기** 후 변경사항이 적용됩니다
- 브라우저 하드 리프레시 (Ctrl+Shift+R) 필요

## 💡 예상되는 문제

만약 위의 모든 항목이 올바르게 설정되어 있는데도 `ApiProjectMapError`가 발생한다면:

1. **결제 계정 미연결**: 가장 흔한 원인입니다
2. **API 활성화 지연**: API를 방금 활성화한 경우 몇 분 더 기다려보세요
3. **API 키 불일치**: `.env.local`의 키가 실제 키와 다른 경우
